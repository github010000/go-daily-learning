# Day 15: 클로저 (Closure)

## 개념 설명
클로저(Closure)는 함수와 그 함수가 선언된 주변 환경(Lexical Environment)의 조합을 의미합니다. Go 언어에서 클로저는 함수 내부에서 정의된 함수가 외부 함수의 변수를 참조하고, 외부 함수의 실행이 종료된 후에도 해당 변수의 상태를 유지할 수 있는 능력을 말합니다.

일반적으로 함수의 지역 변수는 함수 실행이 끝나면 스택에서 사라지지만, 클로저에 의해 참조되는 변수는 힙(Heap) 영역 등에 저장되어 함수가 반환된 이후에도 메모리에 계속 남아있게 됩니다. 이를 통해 함수는 자신만의 '사적인 상태(Private State)'를 가질 수 있습니다.

클로저는 데이터 캡슐화를 구현하는 매우 강력한 도구입니다. 별도의 구조체(struct)를 정의하지 않고도 특정 변수를 외부로부터 숨기면서(Encapsulation), 오직 클로저 함수를 통해서만 해당 변수에 접근하고 수정할 수 있도록 제한할 수 있습니다.

## 코드 설명
1. **`newCounter` 함수**: 
   - `i := 0`이라는 변수를 선언합니다.
   - 익명 함수를 반환하는데, 이 익명 함수는 외부 함수의 `i`를 참조합니다.
   - `counter()`를 호출할 때마다 `i`의 값이 증가하며, 함수가 종료되어도 `i`는 소멸되지 않고 유지됩니다.

2. **`fibonacci` 함수**:
   - 피보나치 수열의 현재 값(`a`)과 다음 값(`b`)을 저장합니다.
   - 반환된 함수를 호출할 때마다 `a, b = b, a+b`를 통해 상태를 업데이트합니다.
   - 이를 통해 함수 호출 시마다 연속적인 수열을 계산할 수 있는 '상태를 가진 생성기' 역할을 합니다.

3. **`makeAdder` 함수**:
   - 함수를 인자로 받아 새로운 함수를 만드는 '함수 팩토리' 패턴입니다.
   - `x`라는 값을 캡처하여, 반환된 함수가 호출될 때마다 입력값 `y`에 `x`를 더하게 만듭니다.

## 핵심 포인트
* **변수 캡처(Variable Capture)**: 클로저는 자신이 선언된 범위의 변수를 기억하고 유지합니다.
* **상태 유지(State Retention)**: 함수 실행이 끝나도 캡처된 변수의 값이 메모리에 유지되어 상태를 이어갈 수 있습니다.
* **데이터 은닉(Encapsulation)**: 외부에서는 클로저 내부의 변수에 직접 접근할 수 없으며, 오직 반환된 함수를 통해서만 제어 가능합니다.
* **함수형 프로그래밍**: 고차 함수(Higher-order function)를 구현하는 핵심 메커니즘입니다.
