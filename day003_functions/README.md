# Day 3: 함수

## 개념 설명

Go의 함수는 프로그램의 기본 구성 요소로, `func` 키워드를 사용해 선언합니다. 함수는 이름, 매개변수 목록, 반환 타입 목록을 가질 수 있으며, 반환값이 없다면 `void` 대신 `func funcName()` 형태로 선언합니다.

Go는 **다중 반환값**을 지원합니다. 하나의 함수에서 여러 값을 동시에 반환할 수 있으며, 이는 특히 오류 처리와 유용하게 쓰입니다 (예: `value, err := someFunc()`). 반환 타입은 괄호로 묶어 나열하고, `return` 문에서 각각의 값을 쉼표로 구분해 반환합니다.

또한, **이름 있는 반환값**(named return values)을 사용하면, 반환값에 이름을 붙여 함수 내에서 변수처럼 사용할 수 있습니다. 이는 코드의 가독성을 높이고 `return` 문을 간결하게 만들어 주지만, 남용 시 가독성을 떨어뜨릴 수 있으므로 주의가 필요합니다.

Go의 함수는 **first-class citizen**으로, 변수에 할당하거나, 다른 함수의 인자/반환값으로 전달될 수 있습니다. 이를 통해 고차 함수(higher-order function)나 함수형 프로그래밍 패턴을 구현할 수 있습니다.

## 코드 설명

- `add(a, b int) int`: 가장 기본적인 함수 선언. 두 정수를 받아 합을 반환.
- `multiReturn(a, b int) (int, int)`: 다중 반환값을 사용하는 예제. 합과 곱을 각각 반환.
- `namedReturn(a, b int) (sum int, product int)`: 이름 있는 반환값 사용. 반환값에 이름을 붙이고 `return`만으로 자동 반환 가능.
- `higherOrderFunc(op func(int, int) int) func(int, int) int`: 함수를 인자로 받고, 함수를 반환하는 고차 함수. `op`을 감싸는 새로운 함수를 반환.
- `main()`: 위 함수들을 실제로 호출하며, 각 특성을 테스트. `wrappedAdd`, `wrappedMultiply`로 고차 함수 활용 예시를 보여 줌.

## 핵심 포인트

- **Go는 다중 반환값을 기본적으로 지원**합니다. 특히 오류 처리(`error` 타입)와 함께 자주 사용됩니다.
- **이름 있는 반환값**은 코드를 간결하게 만드는 도구이지만, 복잡한 로직에서는 명시적 변수 선언이 더 안전할 수 있습니다.
- **함수는 값처럼 취급**되며, 변수에 할당, 인자 전달, 반환 가능 — 이는 함수형 프로그래밍과 콜백 구현의 기반이 됩니다.
- **반환문 생략**: 이름 있는 반환값을 사용할 때는 `return`만으로도 모든 반환값을 자동으로 반환합니다.
