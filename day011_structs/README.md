# Day 11: 구조체(struct)

## 개념 설명

**구조체(struct)**는 서로 연관된 여러 데이터를 하나의 단위로 묶어서 관리할 수 있게 해주는 Go 언어의 핵심적인 사용자 정의 타입입니다. 기본 자료형(int, string 등)만으로는 표현하기 어려운 복잡한 객체의 상태를 표현할 때 사용됩니다.

구조체를 정의할 때는 `type` 키워드와 `struct` 키력드를 사용하며, 내부에는 각 데이터의 이름과 타입을 나타내는 **필드(field)**를 가집니다. 구조체는 단순히 데이터를 담는 그릇 역할을 넘어, 객체 지향 프로그래밍의 데이터 캡슐화와 유사한 역할을 수행합니다.

구조체를 사용할 때는 **구조체 리터럴**을 통해 직관적으로 값을 할당할 수 있습니다. 또한, 구조체의 주소값을 다루는 **포인터**를 활용하면 함수 인자로 구조체를 전달할 때 복사본이 아닌 원본 데이터를 효율적으로 조작할 수 있습니다. Go 언어는 포인터를 통한 필드 접근 시 자동 역참조(Automatic Dereferencing)를 지원하여 문법적 편리함을 제공합니다.

마지막으로, **익명 구조체(Anonymous Struct)**는 이름이 필요 없는 일회성 데이터 구조를 만들 때 사용됩니다. 주로 API 응답을 처리하거나 특정 범위 내에서만 잠시 사용될 데이터를 정의할 때 유용하며, 메모리 효율성과 코드의 간결함을 높여줍니다.

## 코드 설명

1.  **구조체 정의 및 초기화**: `type User struct { ... }`를 통해 새로운 타입을 정의하고, `user1 := User{Name: "홍길동", Age: 25}`와 같이 필드 이름을 명시하여 생성하는 리터럴 방식을 보여줍니다.
2.  **필드 접근 및 수정**: 점(`.`) 연산자를 사용하여 구조체 내부의 필드에 접근하거나 값을 직접 수정하는 방법을 보여줍니다.
3.  **포인터와 자동 역참조**: `&User{...}`를 통해 구조체의 주소값을 생성하고, `user2Ptr.Age`와 같이 포인터를 통해 직접 필드에 접근해도 Go가 알아서 실제 값을 찾아가도록 동작하는 것을 보여줍니다.
4.  **익명 구조체**: 별도의 `type` 선언 없이 `struct { ... }{ ... }` 문법을 사용하여 즉석에서 구조체를 만들고 데이터를 할당하는 방식을 설명합니다.
5.  **구조체 중첩 (Composition)**: `Vehicle` 구조체 내부에 `Car` 구조체를 포함시켜, 복잡한 데이터 계층을 형성하는 방법을 보여줍니다.

## 핵심 포인트

*   **데이터 그룹화**: 연관된 여러 타입을 하나의 논리적 단위로 묶어 관리할 수 있습니다.
*   **구조체 리터럴**: 필드 이름을 명시하여 생성하는 습관을 들이면 코드의 안정성이 높아집니다.
*   **포인터 접근**: 구조체 포인터를 사용하면 원본 데이터를 직접 수정할 수 있으며, 대형 구조체 전달 시 복사 비용을 줄일 수 있습니다.
*   **익명 구조체**: 이름이 불필요한 단발성 데이터 구조(예: JSON 파싱용)에 매우 유용합니다.
*   **자동 역참조**: Go는 포인터를 통한 필드 접근 시 `(*ptr).field` 대신 `ptr.field`를 허용하여 문법을 단순화합니다.
