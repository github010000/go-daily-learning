# Day 36: 테스팅 기초

## 개념 설명
Go 언어에서 소프트웨어 테스트는 표준 라이브러리인 `testing` 패키지를 통해 체계적으로 지원합니다. Go는 컴파일러와 함께 테스트 러너를 기본 제공하여, `_test.go`라는 접미사로 끝나는 파일에 작성된 테스트 코드를 자동으로 감지하고 실행합니다. 테스트 파일은 반드시 테스트 대상 패키지와 동일한 디렉토리에 배치되어야 하며, 파일명은 `예시_test.go` 형태로 작성해야 합니다. 파일 내에서는 `TestXxx` 함수로 시작하는 함수를 정의해야 하며, 이 함수의 첫 번째 인자로 `*testing.T` 구조체를 받아 테스트 로직을 작성합니다.

테스트 실패 처리에는 `t.Error()`와 `t.Fatal()` 두 가지 방식이 있습니다. `t.Error()`는 현재 테스트 케이스를 실패로 표시하지만 테스트 함수의 실행 흐름을 계속 유지합니다. 이는 여러 독립적인 검증 로직이 순차적으로 실행되어야 할 때 유용하며, 한 번의 실행에서 발생할 수 있는 모든 오류를 수집할 수 있게 해줍니다. 반면 `t.Fatal()`은 실패를 기록한 후 즉시 테스트 함수를 종료시킵니다. 이후 코드에 파일 닫기, 연결 해제, 리소스 정리 등이 필요한 경우 `t.Fatal()`이 더 안전하며, 에러가 발생한 시점의 스택 트레이스를 명확하게 보여줍니다.

Go 커뮤니티에서 가장 권장하는 테스트 작성 패턴은 테이블 드리븐 테스트(Table-Driven Tests)입니다. 테스트 케이스를 구조체 슬라이스로 정의하여 입력값(Input), 출력값(Want), 테스트 이름(Name)을 명확히 분리합니다. `for _, tt := range tests` 구문을 사용하여 슬라이스를 순회하며 검증 로직을 반복 실행함으로써, 테스트 코드 중복을 최소화하고 새로운 케이스 추가나 기존 케이스 수정을 매우 용이하게 합니다.

테스트 실행은 터미널에서 `go test` 명령어로 수행합니다. `-v` 플래그는 상세 출력(Verbose)을 활성화하여 각 테스트 함수가 시작하고 끝나는 시점과 로그를 화면에 표시합니다. `-run` 플래그는 정규식을 사용해 특정 테스트 함수만 필터링하여 실행하는 기능입니다. 예를 들어 `go test -v -run=TestFormatString`은 `TestFormatString`으로 시작하는 테스트만 실행하며, 대용량 테스트 스위트에서 특정 기능만 빠르게 검증할 때 필수적으로 사용됩니다.

## 코드 설명
- **테스트 대상 함수 (`Add`, `FormatString`)**: 간단한 비즈니스 로직을 구현한 함수로, 테이블 드리븐 테스트의 검증 대상입니다.
- **테이블 드리븐 테스트 구현**: `tests` 슬라이스에 이름, 입력값, 기대값을 구조체로 정의한 후 `range` 루프로 검증합니다. 각 케이스별 `[성공]` 또는 `[실패]` 메시지를 출력하여 패턴의 동작을 명확히 보여줍니다.
- **t.Error vs t.Fatal 설명**: 실제 `testing` 패키지의 동작 방식을 시뮬레이션하여 설명합니다. 에러 발생 후 계속 진행하는지(`t.Error`), 즉시 중단하는지(`t.Fatal`)의 차이와 사용 시나리오를 정리했습니다.
- **go test -v -run 패턴 설명**: 명령어 옵션의 역할을 안내하고, 실제 개발 환경에서 `*_test.go` 파일을 작성한 후 어떻게 필터링 실행하는지 학습할 수 있도록 구성했습니다.

## 핵심 포인트
- 테스트 파일은 반드시 `_test.go` 접미사를 사용해야 하며, `TestXxx` 함수로 시작해야 `go test`가 인식합니다.
- `t.Error()`는 실패를 기록하지만 실행을 계속하며, `t.Fatal()`은 즉시 종료를 유도합니다. 리소스 정리 필요 시 `Fatal`을 사용하세요.
- 테이블 드리븐 테스트는 구조체 슬라이스와 `range`를 활용해 테스트 케이스를 중앙에서 관리하고 코드 중복을 제거하는 가장 효율적인 Go 테스트 패턴입니다.
- `go test -v`는 상세 출력, `-run=함수이름`은 정규식 필터링 실행을 지원하여 개발 및 CI 파이프라인에서 테스트 범위를 유연하게 제어합니다.
- 본 `main.go`는 개념 이해를 위한 시뮬레이터이며, 실제 프로젝트에서는 `*_test.go` 파일에 `testing` 패키지를 직접 연동하여 실행해야 합니다.
