# Day 27: 제네릭 타입

## 개념 설명
Go 1.18에서 정식 도입된 제네릭(Generic) 타입은 동일한 로직을 다양한 데이터 타입에 적용할 수 있도록 해주는 강력한 언어 기능입니다. 기존에는 `interface{}` 또는 `any`를 사용해 타입을 모호하게 처리하거나, 타입별로 동일한 코드를 반복 작성해야 했지만, 제네릭을 사용하면 컴파일 시점에서 타입을 검증하고 추상화할 수 있습니다. 이는 코드 중복을 크게 줄이고 타입 안전성(Type Safety)을 확보하는 데 핵심적인 역할을 합니다.

제네릭의 핵심 문법은 구조체, 함수, 메서드에 `[T TypeConstraint]` 형식의 타입 매개변수를 추가하는 것입니다. 여기서 `T`는 타입 매개변수 이름이며, `any`는 모든 타입을 허용하는 기본 제약 조건입니다. `golang.org/x/exp/constraints` 패키지는 `Integer`, `Float`, `Ordered`(순서 비교 가능 타입) 등 자주 사용되는 타입 집합을 미리 정의하여 제공합니다. 이를 통해 제네릭 함수나 구조체의 입력 타입을 직관적이고 명시적으로 제한할 수 있습니다.

제네릭을 적용하면 스택(Stack), 큐(Queue), 트리(Tree) 등 데이터 구조를 타입에 독립적으로 구현할 수 있습니다. 예를 들어 `Stack[T any]`로 선언하면 문자열 스택과 정수 스택을 동일한 소스로 생성할 수 있으며, 컴파일러가 타입 불일치를 조기에 잡아주므로 런타임 패닉을 방지하는 데 매우 효과적입니다.

## 코드 설명
- **`Container[T any]`**: `any` 제약 조건을 사용한 제네릭 구조체입니다. 모든 타입의 값을 저장하고 `GetValue()` 메서드를 통해 반환할 수 있습니다.
- **`NumberContainer[T constraints.Integer]`**: `constraints` 패키지의 `Integer` 제약을 적용한 구조체입니다. 이 타입은 정수만 허용하므로, 부동소수점이나 문자열을 대입하면 컴파일 에러가 발생합니다. `Double()` 메서드에서 타입 검증이 컴파일 시점에 이루어집니다.
- **`Stack[T any]`**: 제네릭 스택 구현체입니다. `Push`로 아이템을 추가하고, `Pop`로 제거하며(존재 여부 함께 반환), `IsEmpty`로 상태를 확인합니다. 제네릭 메서드와 슬라이스 기반 구현을 보여줍니다.
- **`Max[T constraints.Ordered]`**: `constraints.Ordered` 제약을 사용하는 제네릭 함수입니다. 정수, 실수 등大小 비교가 가능한 모든 타입에서 동작하며, 타입에 의존하지 않는 알고리즘 구현을 보여줍니다.
- **`main()`**: 각 제네릭 타입과 메서드를 실제 문자열, 정수, 실수 타입에 적용하여 동작을 확인합니다. 제네릭 타입 추론(Type Inference)과 제약 조건 활용을 종합적으로 테스트합니다.

## 핵심 포인트
- 제네릭은 `[T any]` 또는 `constraints` 패키지의 타입 집합을 사용하여 타입 매개변수를 제한하며, 컴파일 시점 타입 검증을 통해 안전성을 보장합니다.
- 구조체와 메서드에 타입 매개변수를 적용하면 타입에 독립적인 재사용 가능한 코드를 작성할 수 있어 코드 중복을 크게 줄일 수 있습니다.
- `golang.org/x/exp/constraints`는 `Integer`, `Float`, `Ordered` 등 자주 쓰이는 타입 집합을 제공하여 제네릭 제약 조건 설정을 직관적으로 만듭니다.
- 제네릭 데이터 구조를 구현하면 동일한 로직으로 다양한 타입의 컬렉션을 타입 안전하게 관리할 수 있으며, 런타임 에러를 사전에 방지할 수 있습니다.
- 제네릭 사용 시 타입 매개변수 이름은 관례적으로 `T`(Type), `K`(Key), `V`(Value)를 사용하며, 제약 조건은 타입 집합 인터페이스로 정의됩니다.
