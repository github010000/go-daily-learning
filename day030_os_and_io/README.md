# Day 30: os & io 패키지

## 개념 설명
Go 언어에서 파일 시스템과 입출력 작업은 `os` 패키지와 `io` 패키지를 통해 강력하고 일관되게 처리됩니다. `os` 패키지는 운영체제 수준의 기능, 예를 들어 파일 생성·열기·읽기·쓰기·삭제, 환경 변수 접근, 그리고 명령줄 인자 처리 등을 제공합니다. 반면 `io` 패키지는 구체적인 구현체와 독립적인 인터페이스(`io.Reader`, `io.Writer`, `io.Seeker` 등)를 정의하여, 파일뿐만 아니라 네트워크 연결, 메모리 버퍼 등 다양한 소스와 대상에 동일한 코드로 입출력을 처리할 수 있도록 추상화합니다.

`os.Open`, `os.Create`, `os.ReadFile`, `os.WriteFile`은 파일 입출력을 위한 대표적인 함수들입니다. `Open`과 `Create`는 파일 핸들을 반환하여 스트림 기반의 점진적인 읽기/쓰기가 가능하도록 하고, `ReadFile`과 `WriteFile`은 메모리에 전체 내용을 로드하거나 메모리에서 직접 파일에 기록하는 간편 함수입니다. 작은 설정 파일이나 템플릿 처리에는 간편 함수가, 대용량 데이터나 실시간 스트리밍 처리에는 스트림 기반 함수가 더 적합합니다.

`bufio.Scanner`는 라인(Line) 단위로 파일을 효율적으로 파싱할 때 사용하는 도구입니다. 내부적으로 버퍼링을 적용하여 성능을 최적화하며, `Scan()` 메서드가 `true`를 반환하는 동안 `Text()`를 호출하면 현재 라인의 내용을 안전하게 얻을 수 있습니다. 또한 `os.Args`는 프로그램 실행 시 전달된 명령줄 인자를 슬라이스로 제공하여, CLI 도구나 설정 옵션을 처리하는 데 필수적으로 활용됩니다.

## 코드 설명
- **os.Args 처리**: `os.Args[0]`로 실행 파일 경로를, `os.Args[1:]`로 전달된 인자를 확인합니다.
- **os.Create & io.Writer**: `os.Create`로 파일을 생성하고 반환된 `*os.File`을 `io.Writer` 인터페이스에 할당합니다. 이를 통해 표준화된 쓰기 인터페이스로 데이터를 작성합니다.
- **os.WriteFile**: 두 번째 줄의 데이터를 덮어쓰기 위해 `os.WriteFile`을 사용합니다. 파일 모드(0644)와 바이트 슬라이스를 전달하여 한 줄로 파일 생성 및 작성을 완료합니다.
- **os.Open & io.Reader**: `os.Open`으로 파일을 열면 `io.Reader` 인터페이스를 구현하므로 `Read` 메서드로 버퍼에 데이터를 읽어올 수 있습니다.
- **bufio.Scanner**: 파일 포인터를 초기화한 후 `bufio.NewScanner`를 생성하고, `for scanner.Scan()` 루프로 라인 단어를 추출하여 출력합니다. `scanner.Err()`으로 파싱 오류를 함께 검사합니다.
- **os.ReadFile**: 마지막으로 `os.ReadFile`를 사용하여 파일 전체 내용을 바이트 슬라이스로 한 번에 로드한 후 문자열로 변환하여 출력합니다.

## 핵심 포인트
- Go는 `io.Reader`와 `io.Writer` 인터페이스를 통해 파일, 네트워크, 버퍼 등 모든 입출력 대상에 통일된 코드를 적용할 수 있습니다.
- `bufio.Scanner`는 라인 단위 처리에 최적화되어 있으며, 대용량 파일에서도 메모리 효율적으로 동작합니다.
- `os.ReadFile`와 `os.WriteFile`은 설정 파일이나 작은 데이터 처리 시 코드를 간결하게 만들어주지만, 대용량 파일에는 스트림 방식(`Open`/`Close`)을 권장합니다.
- `os.Args`는 프로그램 실행 시 전달된 파라미터에 접근하는 표준 방법으로, CLI 개발 시 반드시 숙지해야 합니다.
- 파일 처리 완료 후에는 반드시 `file.Close()`를 호출해야 하며, `defer file.Close()`를 사용하면 예외 상황에서도 안전하게 리소스를 해제할 수 있습니다.
