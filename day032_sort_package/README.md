# Day 32: sort 패키지

## 개념 설명
Go의 `sort` 패키지는 슬라이스 정렬을 위한 표준 라이브러리로, 효율적이고 안전한 정렬 알고리즘을 제공합니다. 내부적으로 TimSort 알고리즘을 기반으로 하며, 이는 합병 정렬과 삽입 정렬을 조합한 안정 정렬(Stable Sort)입니다. 안정 정렬은 키가 동일한 원소의 상대적 순서를 유지하므로, 다중 필드 정렬 시 순서를 보장받을 수 있습니다.

기본 정렬 함수인 `sort.Ints`, `sort.Strings`는 각각 정수 및 문자열 슬라이스를 오름차순으로 정렬할 때 사용합니다. 구조체나 복잡한 타입의 슬라이스를 정렬하려면 `sort.Slice`을 활용하면 함수 리터럴로 정렬 기준을 동적으로 지정할 수 있어 가장 널리 쓰이는 방법입니다.

더 정교한 커스텀 정렬이 필요할 때는 `sort.Interface`를 구현해야 합니다. `Len()`, `Less(i, j int)`, `Swap(i, j int)` 세 가지 메서드를 만족하는 타입은 `sort.Sort()` 함수에 전달할 수 있으며, 이 방식은 정렬 로직을 타입에 캡슐화하고 재사용하기 좋습니다.

`sort.Search` 함수는 이진 탐색(Binary Search)을 수행하여 주어진 조건을 만족하는 첫 번째 인덱스를 반환합니다. 이 함수는 슬라이스가 이미 정렬되어 있고, 조건 함수가 단조 증가(monotonic)한다는 전제 하에 O(log n) 시간 복잡도로 동작합니다. 정렬되지 않은 슬라이스에 사용하면 결과가 정의되지 않으므로 주의가 필요합니다.

## 코드 설명
1. `sort.Ints`와 `sort.Strings`를 사용해 기본 타입 슬라이스의 오름차순 정렬을 구현했습니다.
2. `sort.Slice`을 사용하여 `Student` 구조체 슬라이스를 점수 기준 내림차순으로 정렬했습니다. 함수 리터럴의 `i, j` 인덱스를 통해 비교 로직을 직접 정의합니다.
3. `StudentByScore` 구조체에 `Len`, `Less`, `Swap` 메서드를 구현해 `sort.Interface`를 만족시켰습니다. 이후 `sort.Sort`에 전달해 점수 오름차순 정렬을 적용했습니다.
4. `sort.Search`를 사용해 정렬된 `sortedNums` 슬라이스에서 `target`과 `target2`의 위치를 이진 탐색했습니다. 존재하지 않는 값은 인덱스가 슬라이스 길이를 초과하거나 값이 불일치함을 확인하는 방식으로 처리했습니다.

## 핵심 포인트
- 기본 타입은 `sort.Ints/Strings`, 구조체는 `sort.Slice`을, 복잡한 정렬 로직은 `sort.Interface` 구현을 사용하세요.
- `sort.Slice`과 `sort.Sort`는 모두 안정 정렬을 보장하므로, 동일 키의 순서를 유지해야 할 경우 유용합니다.
- `sort.Search`는 반드시 정렬된 슬라이스에서만 사용해야 하며, 조건 함수는 단조 증가해야 정확한 인덱스를 반환합니다.
- 정렬은 슬라이스 자체를 변경하는 인플레이스(In-place) 연산이므로, 원본 데이터 보존이 필요하면 슬라이스 복사 후 처리하세요.
