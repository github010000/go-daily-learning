# Day 17: 인터페이스(Interface)

## 개념 설명
Go 언어에서 **인터페이스(Interface)**는 타입의 '집합'이 아니라, 타입이 가져야 할 **'행위(Behavior)'**를 정의하는 도구입니다. 인터페이스는 메서드 집합을 정의하며, 어떤 구조체가 그 메서드들을 모두 가지고 있다면 해당 인터페이스를 구현한 것으로 간주됩니다.

Go 인터페이스의 가장 큰 특징은 **암묵적 구현(Implicit Implementation)**입니다. Java나 C#처럼 `implements`라는 키워드를 명시적으로 작성할 필요가 없습니다. 구조체에 인터페이스에 정의된 메서드와 동일한 시그니처의 메서드를 작성하기만 하면, Go 컴파일러가 자동으로 해당 인터페이스를 구현한 것으로 인식합니다. 이를 통해 결합도를 낮추고 유연한 코드 작성이 가능해집니다.

또한, 인터페이스 값은 내부적으로 **(Type, Value)**라는 두 가지 정보를 가집니다. 인터페이스 변수가 `nil`인지 확인하는 것은 매우 중요합니다. 인터페이스 변수가 완전히 `nil`인 상태와, 구체적인 타입 정보는 있지만 값(Value)이 `nil`인 상태는 서로 다릅니다. 이 차이를 이해하는 것이 Go 인터페이스 마스터의 핵심입니다.

## 코드 설명
- **`type Shape interface { Area() float64 }`**: `Area()` 메서드를 반환하는 `float64` 타입을 가진 모든 객체를 `Shape`로 정의합니다.
- **암묵적 구현**: `Rectangle`과 `Circle` 구조체에 `Area()` 메서드를 작성함으로써, 별도의 선언 없이도 `Shape` 인터페이스를 만족하게 됩니다.
- **`printArea(s Shape)`**: 매개변수로 인터페이스 타입을 받음으로써, `Rectangle`이나 `Circle` 중 어떤 것이 들어오더라도 동일한 로직으로 넓이를 출력할 수 있습니다 (다형성).
- **`nil` 인터페이스 체크**: `s == nil`을 통해 인터페이스에 아무런 타입이나 값이 할당되지 않았는지 확인합니다.
- **인터페이스의 내부 구조**: `var rectPtr *Rectangle = nil`을 `s`에 할당하면, `s`는 더 이상 `nil`이 아닙니다. 왜냐하면 `s`는 이제 `*Rectangle`이라는 **타입 정보**를 가지고 있기 때문입니다.

## 핵심 포인트
- **암묵적 구현**: `implements` 키워드 없이 메서드 구현만으로 인터페이스 구현이 완료됩니다.
- **다형성(Polymorphism)**: 동일한 인터페이스를 통해 서로 다른 구조체의 메서드를 호출할 수 있습니다.
- **인터페이스의 구성**: 인터페이스 값은 `(Concrete Type, Concrete Value)`의 쌍으로 이루어집니다.
- **nil 주의사항**: 인터페이스의 타입 정보가 존재하면, 값(Value)이 `nil`이더라도 인터페이스 자체는 `nil`이 아닙니다.
