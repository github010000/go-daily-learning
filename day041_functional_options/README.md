# Day 41: 함수형 옵션 패턴

## 개념 설명
함수형 옵션 패턴(Functional Options Pattern)은 Go 언어에서 구조체의 생성자(Constructor)가 가져야 할 매개변수가 많을 때, 이를 깔끔하게 처리하기 위해 사용하는 디자인 패턴입니다. 일반적으로 객체를 생성할 때 선택적 파라미터(Optional Arguments)가 많으면 생성자를 여러 개 오버로드하거나, 구조체 필드를 모두 채워줘야 하는 번거로움이 발생합니다. 이 패턴은 함수 자체를 옵션으로 전달받아 구조체를 수정하는 방식으로 이를 해결합니다.

이 패턴의 핵심은 `Option`이라는 함수 타입을 정의하는 것입니다. 이 함수 타입은 생성할 구조체의 포인터를 받아서 내부 필드를 수정하는 역할을 합니다. 예를 들어 `WithPort(port int)`와 같은 함수는 정수 값을 받아 해당 필드를 수정하는 함수를 반환합니다. 이를 통해 호출하는 측에서는 마치 설정값을 나열하듯이 가독성 높은 코드를 작성할 수 있습니다.

Rob Pike나 Dave Cheney 등 Go 커뮤니티의 주요 인사들이 권장하는 방식 중 하나로, 특히 외부 라이브러리나 복잡한 설정을 다루는 API를 설계할 때 매우 유용합니다. 호출자는 필요한 설정만 선택적으로 전달할 수 있으므로, API의 호환성을 해치지 않으면서 기능을 확장하기 용이합니다.

## 코드 설명
제공된 코드에서는 `Server` 구조체를 생성하기 위해 이 패턴을 구현한 예제입니다.

1.  **타입 정의**: `type Option func(*Server)`를 통해 Server 포인터를 받는 함수 타입을 정의했습니다.
2.  **옵션 함수 구현**: `WithPort`, `WithHost` 등은 각각 `Option` 타입을 반환합니다. 내부에서는 클로저(Closure)를 사용하여 전달받은 값을 `*Server` 필드에 할당하는 함수를 리턴합니다.
3.  **생성자 설계 (`NewServer`)**: `NewServer` 함수는 `...Option` 가변 인자를 받습니다. 먼저 기본값(Default Value)이 담긴 `Server` 인스턴스를 만들고, 전달된 `options` 슬라이스를 순회하며 각 옵션 함수를 실행합니다. 이렇게 되면 호출자가 전달한 옵션들만 기본값을 덮어씁니다.

## 핵심 포인트
*   **가독성**: `NewServer(WithPort(8080), WithTimeout(5s))`처럼 코드 실행 흐름을 직관적으로 이해할 수 있습니다.
*   **선택적 파라미터**: 필수 매개변수 외의 설정을 쉽게 추가하거나 제거할 수 있습니다.
*   **확장성**: 새로운 설정 항목이 생겨도 기존 코드를 수정하지 않고 `WithXxx` 함수만 추가하면 됩니다.
*   **기본값 처리**: 생성자 내부에서 기본값을 먼저 세팅한 후 옵션을 적용하므로, 누락된 필드는 기본값으로 유지됩니다.
