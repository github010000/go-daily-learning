# Day 9: defer 키워드 학습

## 개념 설명

`defer` 키워드는 Go에서 함수가 반환되기 직전에 특정 함수 호출을 예약하는 역할을 합니다. 이는 마치 '이 함수가 끝나기 직전에 반드시 이 코드를 실행해줘'라고 예약하는 것과 같습니다. 가장 대표적인 사용 사례는 파일 핸들 닫기, 네트워크 연결 종료 등 자원 해제(Cleanup) 패턴입니다.

`defer`로 예약된 함수들은 마치 LIFO(Last-In, First-Out) 스택처럼 작동합니다. 즉, **가장 늦게 `defer`를 호출한 함수가 가장 먼저 실행**되고, 가장 먼저 `defer`를 호출한 함수가 가장 마지막에 실행됩니다. 이 순서를 이해하는 것이 중요합니다.

이러한 특성 덕분에, 함수 본문 실행 중 에러가 발생하여 중간에 리턴되거나(return), 심지어 `panic`으로 인해 비정상 종료되더라도, `defer`로 예약된 정리 로직은 **예외 없이 반드시 실행**됩니다. 이는 Go 프로그램의 안정성과 신뢰성을 높이는 핵심 기법입니다.

## 코드 설명

1. **`cleanupResource` 함수**: 리소스 해제 과정을 시뮬레이션합니다. 실제로 파일을 닫거나 DB 커넥션을 반환하는 로직을 여기에 넣으면 됩니다.
2. **`processData` 함수**: 이 함수는 세 개의 `defer`를 사용합니다. 
   - 첫 번째 `defer`: 가장 먼저 호출된 `defer`이지만, **가장 마지막에 실행**됩니다. 
   - 두 번째 `defer`: 익명 함수와 함께 사용되었으며, `processData`가 종료될 때 **두 번째로 실행**됩니다.
   - **실행 순서**: 함수 본문 실행 $ightarrow$ (defer 2 실행) $ightarrow$ (defer 1 실행).
3. **`main` 함수**: `main` 함수 자체에도 `defer`를 사용하여, 프로그램이 최종적으로 종료될 때 실행되는 전역 정리 로직을 구현했습니다. 또한, 내부의 함수 호출에서 `panic`을 발생시켜도 `defer`가 정상적으로 동작함을 시연합니다.

## 핵심 포인트

* **실행 시점**: `defer`된 함수는 해당 함수가 **반환되거나 (return)**, **패닉(panic)으로 인해 종료될 때** 실행됩니다.
* **실행 순서**: **LIFO (Last-In, First-Out)** 순서를 따릅니다. 가장 늦게 등록된 것이 가장 먼저 실행됩니다.
* **주요 용도**: 파일 닫기 (`defer file.Close()`), DB 트랜잭션 롤백/커밋, 네트워크 연결 종료 등 **자원 해제(Cleanup)** 패턴에 필수적입니다.
* **안정성**: 일반적인 `return`이나 `panic` 상황 모두에서 `defer`는 보장된 실행을 제공하여 프로그램 안정성을 높입니다.
