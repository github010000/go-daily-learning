# Day 22: 채널(Channel)

## 개념 설명
Go 언어에서 **채널(Channel)**은 고루틴(Goroutine) 간에 데이터를 주고받는 통로입니다. 채널은 단순한 데이터 전달 수단을 넘어, 고루틴 간의 **동기화(Synchronization)**를 구현하는 핵심 메커니즘입니다. "메모리를 공유하여 통신하지 말고, 통신을 통해 메모리를 공유하라"는 Go의 철학을 가장 잘 나타내는 도구입니다.

채널은 크게 두 가지 종류가 있습니다. **비버퍼드 채널(Unbuffered Channel)**은 송신자와 수신자가 동시에 준비되어야 데이터 전달이 이루어지며, 한쪽이 준비될 때까지 다른 쪽은 블로킹(Blocking)됩니다. 반면, **버퍼드 채널(Buffered Channel)**은 지정된 크기만큼 데이터를 채널에 쌓아둘 수 있어, 송신자가 수신자의 속도에 즉각적으로 영향을 받지 않고 일정량의 작업을 수행할 수 있게 합니다.

또한, Go는 타입 안정성을 위해 **채널의 방향성**을 지원합니다. 채널을 함수 인자로 넘길 때, 해당 함수가 데이터를 보낼 수만 있는지(`chan<-`), 받을 수만 있는지(`<-chan`), 혹은 양방향인지(`chan`)를 명시할 수 있어 코드의 의도를 명확히 하고 실수를 방지할 수 있습니다.

## 코드 설명
- **`make(chan T)`**: 새로운 채널을 생성합니다. `make(chan int, 3)`과 같이 두 번째 인자를 주면 버퍼 크기가 3인 버퍼드 채널이 생성됩니다.
- **`ch <- val`**: 채널 `ch`에 값 `val`을 송신합니다.
- **`val := <-ch`**: 채널 `ch`로부터 값을 수신하여 변수 `val`에 할당합니다.
- **`chan<- int` (송신 전용)**: 이 타입은 데이터를 채널에 넣을 수는 있지만, 뺄 수는 없습니다. `producer` 함수에서 사용되었습니다.
- **`<-chan int` (수신 전용)**: 이 타입은 데이터를 채널에서 가져올 수는 있지만, 넣을 수는 없습니다. `consumer` 함수에서 사용되었습니다.
- **`close(ch)`**: 채널에 더 이상 보낼 데이터가 없음을 선언합니다. 채널을 닫으면 `range` 문이 종료 조건으로 인식하여 루프를 빠져나옵니다.
- **`for val := range ch`**: 채널이 닫힐 때까지 계속해서 데이터를 수신하며 루프를 돕니다.

## 핵심 포인트
- **Blocking**: 비버퍼드 채널의 송신은 수신자가 준비될 때까지, 수신은 송신자가 데이터를 보낼 때까지 실행을 멈춥니다.
- **Buffer Capacity**: 버퍼가 가득 차면 송신자가 블로킹되고, 버퍼가 비어 있으면 수신자가 블로킹됩니다.
- **Directional Channels**: 함수 파라미터에 채널 방향을 명시함으로써 캡슐화와 타입 안전성을 높일 수 있습니다.
- **Closing**: 채널을 닫는 것은 "데이터 전송이 끝났다"는 신호이며, 닫힌 채널에 다시 데이터를 보내려고 하면 패닉(Panic)이 발생하므로 주의해야 합니다.
