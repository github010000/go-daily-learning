# Day 31: time 패키지

## 개념 설명 섹션
Go의 `time` 패키지는 시간 조회, 지연 실행, 주기적 작업 등 시간 기반 로직을 처리하는 표준 라이브러리입니다. `time.Now()`을 호출하면 현재 시스템 시간을 나타내는 `Time` 구조체가 반환되며, 이를 기반으로 포맷팅이나 파싱 작업을 수행할 수 있습니다. Go에서 시간 간격을 표현하는 `time.Duration`는 내부적으로 `int64` 나노초를 사용하지만, 코드 가독성을 위해 `time.Second`, `time.Millisecond` 같은 단위 상수와 곱셈 연산을 통해 정의하는 것을 권장합니다. `time.Sleep()`은 현재 고루틴을 지정된 시간 동안 강제로 정지시키는 함수로, 간단한 대기 처리에 사용됩니다.

정밀하거나 비동기적인 타이밍 제어가 필요할 때는 `time.Timer`와 `time.Ticker`가 사용됩니다. `Timer`는 설정된 시간 이후에 단 한 번 채널(`C`)로 신호를 보내는 단발성 타이머이며, `Ticker`는 설정된 간격마다 반복적으로 신호를 보내는 반복성 타이머입니다. 둘 다 사용이 끝나면 반드시 `Stop()` 메서드를 호출하여 내부 채널과 타이머 리소스를 해제해야 하며, 특히 `Ticker`는 `defer`를 사용하여 함수 종료 시 자동으로 정리하는 것이 관례입니다.

채널 기반 동시성 패턴과도 자연스럽게 결합됩니다. `time.After()`은 내부적으로 `Timer`를 생성하여 지정된 시간 후 채널로 신호를 보내며, `select` 문과 함께 사용하면 타임아웃 처리를 깔끔하게 구현할 수 있습니다. 시간 문자열 처리는 `time.Parse()`와 `time.Format()`을 통해 수행하며, Go는 포맷팅 기준 시간을 `2006-01-02 15:04:05`로 정해두었습니다. RFC3339나 `time.RFC1123` 같은 표준 상수를 활용하면 이 복잡한 기준을 몰라도 ISO 8601 등 표준 형식을 쉽게 사용할 수 있습니다.

## 코드 설명 섹션
- **1. 현재 시간 및 포맷팅**: `time.Now()`으로 현재 시각을 가져온 후, `Format()` 메서드로 RFC3339 표준 형식과 사용자 정의 형식을 출력하여 `time.Format` 사용법을 보여줍니다.
- **2. Duration와 Sleep**: `500 * time.Millisecond`로 지연 시간을 정의하고 `time.Sleep()`으로 대기한 후, `time.Since()`로 실제 경과 시간을 검증하여 Duration 연산과 Sleep 동작을 설명합니다.
- **3. 시간 파싱 (Parse)**: RFC3339 형식의 문자열을 `time.Parse()`로 파싱하여 `Time` 객체로 변환한 뒤, 연/월/일/시/분/초 접근자를 통해 추출하는 방법을 보여줍니다.
- **4. Timer**: `time.NewTimer(2s)`로 2초 후 신호를 보내는 타이머를 생성하고, `select`를 통해 채널 신호를 받으면 타이머를 종료합니다. `Stop()` 호출로 리소스 누수를 방지하는 패턴을 설명합니다.
- **5. Ticker와 After**: 1초 주기의 `Ticker`와 3초 지연의 `After` 채널을 `select`에서 동시에 모니터링합니다. 3초가 지나면 `After`가 신호를 보내 루프를 종료하고, 실행 횟수를 출력하여 채널 기반 타이밍 제어 방식을 보여줍니다.

## 핵심 포인트 섹션
- `time.Duration`는 나노초 단위 정수형이며, `time.Second * 2` 형태로 단위 상수를 곱해서 사용하는 것이 Go의 표준 관례입니다.
- `Timer`와 `Ticker`는 반드시 `Stop()`을 호출해야 하며, `Ticker`는 `defer ticker.Stop()`을 사용해 함수 종료 시 자동으로 정리하는 것이 권장됩니다.
- `time.After()`은 채널 기반 타임아웃 구현에 최적화되어 있으며, `select` 문과 결합하면 복잡한 타임아웃 로직을 깔끔하게 처리할 수 있습니다.
- 시간 포맷팅 기준은 `2006-01-02 15:04:05`이며, `time.RFC3339` 상수를 활용하면 표준 ISO 8601 형식을 손쉽게 사용할 수 있습니다.
