# Day 18: 타입 단언과 타입 스위치

## 개념 설명

Go 언어에서 `interface{}`(또는 최근 버전의 `any`)는 어떤 타입의 값이라도 담을 수 있는 유연한 타입을 제공합니다. 하지만 interface 타입에 저장된 값은 런타임 전에는 그 실제 구체적인 타입을 알 수 없습니다. 따라서 interface에 담긴 값을 원래의 구체적인 타입(string, int 등)으로 되돌려 사용하기 위해서는 **타입 단언(Type Assertion)**이 필요합니다.

**타입 단언**은 interface 변수가 특정 타입이라고 선언하는 것입니다. 만약 단언한 타입과 실제 데이터의 타입이 일치하지 않으면 프로그램은 즉시 런타임 패닉(Panic)을 일으키며 종료됩니다. 이러한 위험을 방지하기 위해 Go는 두 번째 반환값으로 성공 여부를 알려주는 **안전한 단언(Comma-ok idiom)** 방식을 제공합니다.

**타입 스위치(Type Switch)**는 단언을 여러 번 반복하는 대신, 하나의 interface 변수가 가질 수 있는 여러 후보 타입들을 깔끔하게 분기 처리할 수 있는 문법적 설탕(Syntactic Sugar)입니다. `switch v := i.(type)` 구문을 사용하면 각 `case` 문 안에서 변수 `v`가 해당 케이스의 구체적인 타입으로 자동 변환되어 매우 편리하게 다형성을 구현할 수 있습니다.

## 코드 설명

- **`i.(string)`**: 가장 기본적인 타입 단언입니다. `i`가 `string`임을 확신할 때 사용하지만, 실패 시 프로그램이 멈출 위험이 있습니다.
- **`val, ok := i.(int)`**: 안전한 단언 방식입니다. `ok`라는 boolean 변수를 통해 타입 일치 여부를 확인하므로, `if !ok` 문을 통해 에러 처리를 안전하게 할 수 있습니다.
- **`switch v := i.(type)`**: 타입 스위치의 핵심 구문입니다. `i.(type)`을 통해 interface 내부의 타입을 검사하고, `v`라는 변수에 해당 타입으로 변환된 값을 할당하여 `case` 블록 내에서 즉시 해당 타입의 기능을 사용할 수 있게 합니다.
- **`printInterfaceValue` 함수**: 다형성 처리 패턴의 전형적인 예시입니다. 입력받은 `interface{}`의 타입에 따라 문자열 길이를 구하거나, 정수를 2배로 만드는 등 서로 다른 로직을 실행합니다.

## 핵심 포인트

- **타입 단언의 위험성**: 잘못된 타입 단언은 런타임 패닉을 유발하므로 주의가 필요합니다.
- **안전한 단언(Comma-ok)**: `v, ok := i.(T)` 패턴을 사용하여 예외 상황을 우아하게 처리하는 습관을 들여야 합니다.
- **타입 스위치의 강력함**: `switch v := i.(type)` 구문은 `case` 내부에서 변수 `v`의 타입을 해당 타입으로 확정시켜주므로 별도의 단언 작업 없이 로직을 수행할 수 있습니다.
- **다형성 구현**: 인터페이스와 타입 스위치를 조합하면 다양한 데이터 타입을 동일한 함수로 처리하는 강력한 다형성 패턴을 만들 수 있습니다.
