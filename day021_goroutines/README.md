# Day 21: 고루틴(Goroutine)

## 개념 설명

**고루틴(Goroutine)**은 Go 언어의 가장 강력한 특징 중 하나로, 매우 가볍고 효율적인 경량 스레드(Lightweight Thread)입니다. 일반적인 OS 스레드는 생성 시 수 MB의 스택 메모리를 할당받고 컨텍스트 스위칭 비용이 크지만, 고루틴은 실행 시 수 KB의 아주 작은 스택에서 시작하며 필요에 따라 동적으로 크기를 조절합니다.

고루틴을 실행할 때는 함수 호출 앞에 `go` 키워드만 붙이면 됩니다. 이를 통해 프로그램은 현재 실행 중인 흐름을 멈추지 않고, 새로운 작업을 백그라운드에서 병렬 또는 병행적으로 처리할 수 있습니다. 수만, 심지어 수백만 개의 고루틴을 동시에 생성해도 OS 스레드에 비해 시스템 부하가 매우 적습니다.

하지만 주의할 점이 있습니다. 메인 고루틴(Main Goroutine)은 다른 고루틴들이 작업 중이더라도 자신의 코드가 끝나면 즉시 프로그램을 종료합니다. 따라서 실행 중인 고루틴들이 작업을 마칠 때까지 기다려주는 **동기화 메커니즘**이 반드시 필요합니다. 이때 주로 사용하는 것이 `sync.WaitGroup`입니다.

또한, **고루틴 누수(Goroutine Leak)**를 주의해야 합니다. 고루틴이 무한 루프에 빠지거나, 채널(Channel)로부터 데이터를 받기 위해 영원히 대기 상태에 머물게 되면, 프로그램이 종료될 때까지 메모리를 점유하며 사라지지 않습니다. 이는 메모리 사용량을 점진적으로 증가시켜 결국 시스템 장애를 유발할 수 있습니다.

## 코드 설명

- **`go task(...)`**: `go` 키워드는 뒤에 오는 함수를 새로운 고루틴으로 생성하여 비동기적으로 실행하도록 명령합니다.
- **`sync.WaitGroup`**:
    - `wg.Add(n)`: 완료될 작업의 개수를 설정합니다.
    - `wg.Done()`: 하나의 작업이 완료되었음을 알립니다(내부 카운트를 1 감소).
    - `wg.Wait()`: 모든 작업이 완료되어 카운트가 0이 될 때까지 메인 함수를 대기시킵니다.
- **`time.Sleep(duration)`**: 각 고루틴이 각기 다른 시간 동안 작업 중임을 시뮬레이션하여, 순차 실행과 병렬 실행의 시간 차이를 명확히 보여줍니다.
- **`defer wg.Done()`**: 함수가 정상적으로 끝나거나 에러로 인해 종료되더라도 반드시 카운트를 감소시켜 `Wait()`이 무한 대기에 빠지는 것을 방지합니다.

## 핵심 포인트

- **경량성**: 고루틴은 OS 스레드보다 훨씬 적은 메모리를 사용하며 생성 비용이 매우 저렴합니다.
- **비동기 실행**: `go` 키워드 하나로 복잡한 멀티스레딩 로직을 단순하게 구현할 수 있습니다.
- **동기화 필수**: 고루틴의 생명주기를 관리하기 위해 `sync.WaitGroup`이나 `channel`을 통한 제어가 필수적입니다.
- **누수 방지**: 종료되지 않는 고루틴이 발생하지 않도록 리소스 해제와 종료 조건을 명확히 설계해야 합니다.
