# Day 23: select 문 - 다중 채널 제어와 타임아웃 패턴

## 개념 설명
Go 언어의 `select` 문은 채널을 위한 `switch` 문이라고 이해할 수 있습니다. 여러 개의 채널 작업을 동시에 대기하고, 그중 가장 먼저 준비된(데이터를 보낼 수 있거나 받을 수 있는) 케이스를 실행하는 제어 구조입니다. 이는 병렬로 동작하는 여러 고루틴의 응답을 효율적으로 관리할 때 필수적입니다.

`select` 문은 기본적으로 'Blocking' 방식으로 동작합니다. 즉, `case`에 명시된 채널 중 어느 것도 준비되지 않았다면 프로그램은 실행을 멈추고 기다립니다. 하지만 `default` 케이스를 추가하면 채널이 준비되지 않았을 때 기다리지 않고 즉시 다음 코드를 실행하는 'Non-blocking' 구현이 가능합니다.

또한, 네트워크 통신이나 데이터 처리에서 응답이 너무 늦어질 경우를 대비해 `time.After`와 함께 사용하여 '타임아웃(Timeout) 패턴'을 구현할 수 있습니다. 이는 시스템의 안정성을 높이고 무한 대기로 인한 데드락(Deadlock)을 방지하는 데 매우 중요한 역할을 합니다.

## 코드 설명
- **다중 채널 대기 (`ch1`, `ch2`)**: `ch1`은 1초, `ch2`는 3초 후에 데이터를 보냅니다. `select` 문은 `time.After(2 * time.Second)`와 함께 사용되어, 2초가 지나면 `ch2`가 준비되기 전에 타임아웃 케이스를 먼저 실행합니다.
- **타임아웃 패턴 (`time.After`)**: `time.After`는 지정된 시간이 지나면 채널에 현재 시간을 전송합니다. 이를 `select`의 `case`로 사용하면 특정 작업이 정해진 시간 내에 완료되지 않았을 때 예외 처리를 할 수 있습니다.
- **Non-blocking (`default`)**: `unbufferedCh`는 데이터가 들어있지 않은 상태입니다. 이때 `default`가 없는 `select`는 데이터가 올 때까지 무한히 기다리지만, `default`가 있으면 즉시 "데이터가 준비되지 않아..." 메시지를 출력하고 넘어갑니다.
- **준비된 채널 처리**: 버퍼가 있는 채널(`readyCh`)에 미리 데이터를 넣어두면, `select` 문이 실행되자마자 기다림 없이 즉시 해당 `case`를 실행합니다.

## 핵심 포인트
- **랜덤 선택**: 만약 여러 개의 `case`가 동시에 준비된 상태라면, Go 런타임은 이 중 하나를 **무작위(Random)**로 선택합니다. (결정론적이지 않음)
- **Non-blocking 구현**: `default` 케이스를 사용하면 채널의 준비 여부와 상관없이 코드를 계속 진행할 수 있습니다.
- **타임아웃 관리**: `time.After`를 사용하여 응답 없는 고루틴으로 인해 전체 시스템이 멈추는 것을 방지합니다.
- **Deadlock 방지**: `select`는 채널 작업의 흐름을 제어하여 고루틴 간의 의존성을 안전하게 관리하게 해줍니다.
