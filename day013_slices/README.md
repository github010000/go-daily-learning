# Day 13: 슬라이스(Slice) 마스터하기

## 개념 설명

Go 언어에서 **슬라이스(Slice)**는 배열의 요소를 가리키는 참조 타입(Reference Type)의 데이터 구조입니다. 배열과 달리 슬라이스는 크기가 동적으로 변할 수 있어, Go 프로그래밍에서 가장 빈번하게 사용되는 컬렉션 타입입니다.

슬라이스는 내부적으로 세 가지 정보를 담고 있는 작은 구조체입니다:
1. **포인터(Pointer):** 실제 데이터가 저장되어 있는 하위 배열(Underlying Array)의 시작 주소.
2. **길이(Length):** 슬라이스가 현재 가지고 있는 요소의 개수.
3. **용량(Capacity):** 슬라이스의 시작 주소부터 하위 배열의 끝까지 포함된 요소의 총 개수.

슬라이스를 다룰 때 주의할 점은 슬라이스의 요소를 수정하면 슬라이스가 참조하고 있는 하위 배열의 값이 직접 변경된다는 점입니다. 이는 슬라이스가 배열을 공유하는 '뷰(View)' 역할을 하기 때문입니다.

## 코드 설명

1.  **배열 참조 (`arr[1:4]`)**: 배열의 특정 범위를 슬라이스로 추출하면, 새로운 배열이 생기는 것이 아니라 기존 배열의 메모리 영역을 가리키게 됩니다. 따라서 슬라이스 값을 바꾸면 원본 배열도 바뀝니다.
2.  **`make` 함수**: `make([]int, 3, 5)`와 같이 사용하면, 길이는 3이지만 용량은 5인 슬라이스를 생성합니다. 이는 초기에는 3개만 사용하지만, `append`를 통해 5개까지는 메모리 재할당 없이 확장 가능하다는 의미입니다.
3.  **`append` 함수**: 슬라이스에 새로운 요소를 추가합니다. 만약 슬라이스의 `len`이 `cap`에 도달하면, Go는 더 큰 새로운 배열을 생성하고 기존 내용을 복사한 뒤 새로운 주소를 할당합니다.
4.  **`copy` 함수**: 한 슬라이스의 내용을 다른 슬라이스로 복사합니다. 이때 복사되는 양은 대상(destination) 슬라이스의 길이에 의해 결정되므로 주의가 필요합니다.
5.  **Nil vs Empty Slice**: `var s []int`는 `nil`을 가리키며 메모리 할당이 전혀 없는 상태입니다. 반면 `s := []int{}`는 길이가 0인 슬라이스를 위한 메모리가 할당된 상태입니다.

## 핵심 포인트

*   **참조 타입**: 슬라이스는 데이터를 직접 들고 있는 것이 아니라 배열의 주소를 참조합니다.
*   **Length vs Capacity**: `len`은 현재 유효한 데이터 개수, `cap`은 재할당 없이 늘릴 수 있는 한계치입니다.
*   **배열 공유**: 슬라이스를 슬라이싱(`[:]`)하면 동일한 하위 배열을 공유하므로 side-effect에 주의해야 합니다.
*   **동적 확장**: `append` 사용 시 용량이 부족하면 새로운 배열이 생성되며, 이때 기존 슬라이스의 포인터가 변경될 수 있습니다.
