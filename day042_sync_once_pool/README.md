# Day 42: sync.Once & sync.Pool

## 개념 설명
Go 언어의 `sync` 패키지에는 동시성 프로그래밍을 돕는 두 가지 매우 유용한 구조체가 있습니다. 첫 번째는 **`sync.Once`**입니다. 이는 `Once.Do` 메서드를 호출하여 특정 함수를 단 한 번만 실행하도록 보장하는 기능입니다. 주로 글로벌 변수 초기화, 설정 로드, 데이터베이스 연결과 같은 중복 비용이 큰 작업을 여러 goroutine에서 안전하게 실행해야 할 때 사용됩니다. 두 번째는 **`sync.Pool`**입니다. 이는 임시 객체를 저장하고 재사용하여 GC(Garbage Collector)의 부하를 줄이는 객체 풀(Object Pool) 패턴을 구현한 것입니다. 객체를 생성하고 소멸하는 과정은 메모리 관리 측면에서 비용이 발생할 수 있으므로, 자주 사용되는 객체를 풀에 미리 넣어두고 `Get`, `Put`으로 재사용함으로써 성능을 향상시킬 수 있습니다.

## 코드 설명
- **Sync.Once 구현**: `Singleton` 타입의 초기화 로직을 `once.Do` 블록 안에 감쌌습니다. `main` 함수에서 `NewInstance()`를 두 번 호출해도 `[Sync.Once] 초기화 로직 수행` 메시지는 단 한 번만 출력됩니다. 이는 여러 goroutine에서 동시에 호출되어도 안전합니다.
- **Sync.Pool 구현**: `PoolType`을 관리하는 `sync.Pool`을 정의했습니다. `New` 함수는 풀에 저장된 객체가 없을 때만 새 객체를 생성하도록 했습니다.
- **Get/Put 동작**: 먼저 `Get`을 통해 객체를 가져와 사용하고, `Put`으로 다시 풀에 넣습니다. 이후 다시 `Get`할 때 `Pool.New`가 호출되지 않고 반납되었던 객체가 재사용되는 것을 확인할 수 있습니다.

## 핵심 포인트
- `sync.Once`는 함수의 실행을 보장하기 위한 전역 동기화 수단으로, 초기화 비용이 큰 작업을 보호합니다.
- `sync.Pool`의 `Get`은 풀에 객체가 있으면 즉시 반환하고, 없으면 `New` 함수를 호출하여 새 객체를 생성합니다.
- 풀에서 사용한 객체는 `Put`으로 반드시 반납해야 하며, 풀 밖으로 반납된 객체는 재사용될 수 없습니다.
- `sync.Pool`은 GC 압력을 줄일 수 있지만, 객체의 상태(State) 관리가 복잡해질 수 있으므로 주의가 필요합니다.
- `sync.Pool`의 수명은 명확하지 않으며, Go 런타임이 필요에 따라 풀의 객체를 정리할 수 있습니다.
