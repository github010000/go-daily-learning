# Day 16: 메서드(Method)

## 개념 설명
Go 언어에서 **메서드(Method)**는 특정 타입에 종속된 함수를 의미합니다. 일반적인 함수와 달리, 메서드는 '수신자(Receiver)'를 가짐으로써 특정 데이터 타입에 특수한 동작을 부여할 수 있습니다. 이를 통해 객체 지향 프로그래록(OOP)의 캡슐화와 유사한 효과를 낼 수 있습니다.

메서드는 크게 **값 수신자(Value Receiver)**와 **포인터 수신자(Pointer Receiver)**로 나뉩니다. 값 수신자는 메서드 호출 시 데이터의 복사본을 전달받으므로 원본 데이터를 변경할 수 없습니다. 반면, 포인터 수신자는 데이터의 메모리 주소를 전달받으므로 메서드 내부에서 원본 데이터를 직접 수정할 수 있으며, 대용량 구조체의 경우 복사 비용을 줄이는 효율성도 얻을 수 있습니다.

또한, Go의 독특한 특징 중 하나는 구조체(struct)뿐만 아니라, `type MyInt int`와 같이 기본 타입을 기반으로 정의한 **사용자 정의 타입**에도 메서드를 추가할 수 있다는 점입니다. 이는 언어의 유연성을 극대화하며, 기본 타입에 새로운 행위(Behavior)를 부여할 수 있게 합니다.

## 코드 설명
- `func (r Rectangle) Area() float64`: **값 수신자** 예제입니다. `Rectangle`의 복사본을 사용하여 면적을 계산하며, 원본 `rect`의 값은 변하지 않습니다.
- `func (r *Rectangle) Scale(factor float64)`: **포인터 수신자** 예제입니다. `*Rectangle`을 수신자로 사용하여 메서드 내부에서 `width`와 `height`를 직접 수정합니다.
- `type MyInt int`: 구조체가 아닌 기본 정수형을 기반으로 새로운 타입을 선언했습니다.
- `func (m MyInt) IsEven() bool`: **비구조체 타입에 대한 메서드** 예제입니다. `MyInt`라는 타입에 짝수 여부를 판별하는 기능을 추가했습니다.
- `main` 함수: 각 메서드가 어떻게 동작하는지(값의 변화 유무, 타입별 동작)를 출력하여 시각적으로 확인할 수 있게 구성되었습니다.

## 핵심 포인트
- **수신자(Receiver) 정의**: `func (r ReceiverType) MethodName()` 형식을 사용합니다.
- **값 수신자 vs 포인터 수신자**:
    - 원본 수정이 필요하거나 성능 최적화(복사 방지)가 필요하면 **포인터 수신자**를 사용합니다.
    - 원본을 보호하고 읽기 전용 동작만 수행하려면 **값 수신자**를 사용합니다.
- **확장성**: 구조체뿐만 아니라 기본 타입(int, string 등)을 기반으로 만든 모든 사용자 정의 타입에 메서드를 붙일 수 있습니다.
- **자동 참조**: Go는 메서드 호출 시 상황에 따라 포인터나 값으로의 자동 변환(Automatic Dereferencing/Referencing)을 지원하여 편리한 호출이 가능합니다.
