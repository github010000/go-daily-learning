# Day 10: 포인터 (Pointers)

## 개념 설명

프로그래밍에서 포인터는 '메모리 주소'를 저장하는 변수입니다. 일반 변수가 실제 데이터를 저장하는 공간에 값을 저장한다면, 포인터 변수는 그 데이터가 **실제로 저장된 위치(주소)**를 저장합니다. 이는 C/C++에서 흔히 접하는 개념이지만, Go에서는 메모리 관리를 추상화하여 제공하는 만큼, C/C++만큼 저수준에서 직접 메모리를 조작할 일은 거의 없습니다.

Go에서 포인터를 다루기 위해 반드시 알아야 할 핵심 연산자가 바로 **주소 연산자(`&`)**와 **역참조 연산자(`*`)**입니다. `&` 연산자는 변수 앞에 붙여 해당 변수의 메모리 주소를 얻어 포인터 변수에 저장하는 역할을 합니다. 반면, `*` 연산자는 포인터 변수가 담고 있는 주소로 찾아가서 그곳에 저장된 **실제 값**에 접근하거나 값을 변경할 때 사용됩니다. 

포인터를 사용하는 가장 중요한 이유는 '값 전달'과 '주소 전달'의 차이점을 이해하기 위해서입니다. Go에서 함수에 변수를 인자로 넘기면 기본적으로 '값 복사'가 일어나므로, 함수 내부에서 변경해도 원본 변수는 영향을 받지 않습니다. 만약 함수 내부에서 원본 변수 자체를 수정해야 할 필요가 있다면, 그 변수의 **주소(포인터)**를 넘겨주어야 합니다.

이러한 포인터 전달 방식(Pass by Pointer)을 사용하면, 함수를 호출한 곳의 메모리 위치에 직접 접근하여 상태를 영구적으로 변경(Side Effect)할 수 있게 됩니다. 예제 코드는 이 두 가지 전달 방식의 차이를 명확하게 보여줍니다.

## 코드 설명

1. **`&originalValue` (주소 연산자)**: `&`를 사용하여 `originalValue` 변수가 메모리의 어느 주소에 저장되었는지 그 주소를 가져옵니다. 이 주소를 `ptr`이라는 포인터 변수에 저장합니다.
2. **`*ptr` (역참조 연산자)**: `*`를 사용하여 `ptr`이 가지고 있는 주소로 이동한 후, 그 주소에 저장된 실제 값(예: 100)을 읽어옵니다. 만약 `*ptr = 200`과 같이 사용하면, 그 주소에 저장된 값을 200으로 덮어쓰게 됩니다.
3. **`incrementByValue(valueCopy)`**: 이 함수는 `int` 타입의 **값 복사본**을 받습니다. 함수 내부에서 `x = x + 10`을 해도, 이는 스택에 존재하는 `x`라는 임시 변수의 값만 변경할 뿐, `main` 함수의 `valueCopy`를 변경하지 못합니다.
4. **`incrementByPointer(p *int)`**: 이 함수는 `*int` 타입의 **포인터**를 받습니다. 함수 내부에서 `*p = *p + 100`을 사용하면, 포인터 `p`가 가리키는 원본 메모리 주소(`originalValue`의 주소)에 직접 접근하여 그 값을 100 증가시킵니다. 이 변경은 함수가 끝난 후에도 `main` 함수에 남아있게 됩니다.

## 핵심 포인트

*   **`&` (주소 연산자):** 변수 앞에 붙여 '이 변수의 메모리 주소'를 얻습니다. (예: `&x`) 
*   **`*` (역참조 연산자):** 포인터 변수 앞에 붙여 '이 주소에 저장된 실제 값'을 읽거나 쓰기 합니다. (예: `*p`) 
*   **값 전달 vs. 포인터 전달:** 값 전달은 복사본만 변경하고 원본은 그대로 두지만, 포인터 전달은 원본 메모리 위치를 직접 수정할 수 있게 합니다.
*   **Go의 장점:** Go는 포인터 연산을 매우 명시적으로 관리하여, 개발자가 메모리 누수나 잘못된 메모리 접근(댕글링 포인터 등)을 할 위험을 줄여줍니다.
