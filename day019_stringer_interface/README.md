# Day 19: Stringer & fmt 인터페이스

## 개념 설명

Go 언어의 `fmt` 패키지에는 `Stringer`라는 매우 중요한 인터페이스가 정의되어 있습니다. 이 인터페이스의 구조는 매우 단순합니다. 바로 `String() string`이라는 메서드를 하나 가지고 있는 형태입니다.

`fmt.Println`, `fmt.Printf`, `fmt.Sprintf`와 같은 표준 출력 함수들은 출력하려는 값이 `fmt.Stringer` 인터페이스를 구현하고 있는지 확인합니다. 만약 해당 구조체가 `String() string` 메서드를 가지고 있다면, `fmt` 패키지는 구조체의 내부 필드를 그대로 보여주는 대신, `String()` 메서드가 반환하는 문자열을 사용하여 값을 출력합니다.

이 기능을 활용하면 복잡한 구조체를 사람이 읽기 좋은(Human-readable) 형태로 변환할 수 있습니다. 예를 들어, 날짜 형식을 특정 포맷으로 맞추거나, 로그를 남길 때 구조체의 핵심 정보만 요약해서 보여줄 수 있습니다.

또한, 보안적인 측면에서도 매우 유용합니다. 구조체 내부에 비밀번호, API 키, 개인정보와 같은 민감한 데이터가 포함되어 있을 때, `String()` 메서드를 커스텀하게 구현하여 해당 필드가 출력되지 않도록 제어할 수 있습니다. 이는 실수로 로그 파일에 민감 정보가 남는 것을 방지하는 강력한 방어 기제가 됩니다.

## 코드 설명

1.  **`Person` 구조체 (미구현)**: `String()` 메서드를 구현하지 않았습니다. 따라서 `fmt.Printf("%+v", person)`를 사용하면 구조체의 필드명과 값이 기본 형태(`{홍길동 30}`)로 출력됩니다.
2.  **`Product` 구조체 (구현)**: `String() string` 메서드를 구현했습니다. 이 메서드 내부에서 `fmt.Sprintf`를 사용하여 상품 ID, 이름, 가격을 보기 좋게 포맷팅하여 반환합니다. 결과적으로 `fmt.Println(product)` 호출 시 우리가 정의한 문자열이 출력됩니다.
3.  **`Admin` 구조체 (보안 활용)**: `Password` 필드를 가지고 있지만, `String()` 메서드에서 이 필드를 제외하고 출력하도록 로직을 작성했습니다. 이를 통해 `fmt.Println`을 사용할 때 실수로 비밀번호가 노출되는 것을 원천 차단합니다.
4.  **슬라이스 출력**: `[]Product` 슬라이스를 반복문으로 출력할 때, 각 요소가 `Stringer`를 구현했기 때문에 별도의 포맷팅 없이도 깔끔한 출력이 가능합니다.

## 핵심 포인트

*   **인터페이스 정의**: `fmt.Stringer`는 `String() string` 메서드 하나로 구성된 인터페이스입니다.
*   **자동 호출**: `fmt` 패키지의 출력 함수들은 전달된 인자가 `Stringer`를 구현했는지 내부적으로 체크하고, 구현했다면 `String()`을 자동 호출합니다.
*   **가독성 향상**: 구조체의 복잡한 데이터를 개발자가 원하는 깔끔한 문자열로 변환하여 디버깅을 용이하게 합니다.
*   **보안 및 캡슐화**: 로그 출력 시 민감한 데이터를 마스킹하거나 제외하여 보안 사고를 예방할 수 있습니다.
