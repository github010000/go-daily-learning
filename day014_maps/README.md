# Day 14: 맵(Map)

## 개념 설명
**맵(Map)**은 Go 언어에서 제공하는 **키(Key)와 값(Value)의 쌍**으로 이루어진 데이터 구조입니다. 다른 언어에서는 해시 테이블(Hash Table) 또는 딕셔너리(Dictionary)라고 불리기도 합니다. 맵은 특정 키를 통해 매우 빠르게 값에 접근할 수 있다는 강력한 장점이 있습니다.

맵을 사용할 때 가장 주의해야 할 점은 **초기화**입니다. 단순히 `var m map[string]int`와 같이 선언만 한 상태는 `nil` 맵이며, 이 상태에서 데이터를 추가하려고 하면 프로그램이 충돌(panic)하며 종료됩니다. 따라서 `make` 함수를 사용하여 메모리를 할당하거나, 맵 리터럴을 통해 초기화된 상태로 사용해야 합니다.

또한, 맵은 **순서가 보장되지 않는** 자료구조입니다. 동일한 데이터를 넣고 순회하더라도 실행할 때마다 출력되는 순서가 달라질 수 있으므로, 순서가 중요한 작업이 필요하다면 별도의 슬라이스(Slice) 등을 함께 관리해야 합니다.

## 코드 설명
- **맵 리터럴 초기화**: `map[string]int{...}` 형식을 사용하여 선언과 동시에 초기 데이터를 삽입합니다.
- **make 함수**: `make(map[KeyType]ValueType)`를 통해 빈 맵을 안전하게 생성합니다.
- **Comma ok idiom**: `value, ok := m[key]` 패턴을 사용하여, 키가 맵에 존재하는지 여부를 불리언(`bool`) 값으로 확인합니다. 이는 존재하지 않는 키를 참조했을 때 기본값(0, "")이 반환되는 것과 실제 키가 없는 것을 구분하기 위해 매우 중요합니다.
- **delete 함수**: `delete(map, key)`를 통해 특정 요소를 삭제합니다. 삭제하려는 키가 없어도 에러가 발생하지 않습니다.
- **for range 루프**: 맵의 모든 요소를 순회하며 키와 값을 하나씩 꺼내어 처리합니다.

## 핵심 포인트
- 맵의 키(Key)는 반드시 **비교 가능한(comparable) 타입**이어야 합니다 (예: string, int, float, bool 등).
- 맵은 **참조 타입(Reference Type)**이므로, 함수 인자로 전달될 때 복사가 아닌 원본 맵을 가리키게 됩니다.
- **nil 맵**에 값을 쓰려고 하면 런타임 에러가 발생하므로 항상 `make`나 리터럴로 초기화하세요.
- 맵의 요소 순회 순서는 **예측 불가능(Random)** 하다는 점을 명심하세요.
