# Day 12: 배열 (Array)

## 개념 설명
배열(Array)은 동일한 타입의 데이터를 메모리 상에 연속적으로 나열한 자료구조입니다. Go 언어에서 배열은 가장 기본적인 데이터 집합체 중 하나로, 선언 시 반드시 그 **크기(Length)**를 지정해야 합니다.

배열의 가장 큰 특징은 **고정된 길이**를 가진다는 점입니다. 한 번 생성된 배열의 크기는 프로그램 실행 중에 변경할 수 없습니다. 따라서 배열의 크기는 단순한 속성이 아니라, 해당 데이터 타입의 **일반적인 구성 요소(Type Identity)**가 됩니다. 예를 들어, `[3]int`와 `[4]int`는 서로 다른 타입으로 취급됩니다.

또한, Go에서 배열은 **값 타입(Value Type)**입니다. 이는 배열을 변수에 할당하거나 함수에 인자로 전달할 때, 참조(주소)가 아닌 배열 내부의 모든 요소가 새로운 메모리 공간에 복사된다는 것을 의미합니다. 이러한 특성은 데이터의 무결성을 보장할 수 있지만, 매우 큰 배열을 다룰 때는 메모리 복사 비용을 고려해야 합니다.

## 코드 설명
- **배열 선언 및 초기화**: `var arr [5]int`와 같이 크기를 명시하여 선언하거나, `[...]int{...}`를 사용하여 컴파일러가 요소의 개수를 보고 크기를 결정하게 할 수 있습니다.
- **다차원 배열**: `[2][2]int`와 같이 배열의 요소로 또 다른 배열을 포함시켜 행렬 형태의 구조를 만들 수 있습니다.
- **값 타입(Value Type) 실험**: `copyArr := original` 코드를 통해 원본 배열의 값이 어떻게 복사되는지 보여줍니다. `copyArr[0]`을 수정해도 `original[0]`은 변하지 않음을 통해 배열이 참조가 아닌 '값'임을 증명합니다.
- **타입 비교**: `[3]int`와 `[4]int`를 각각 출력하여, 크기가 다르면 서로 다른 타입(`%T`)으로 인식됨을 확인합니다.

## 핵심 포인트
- **고정 크기**: 배열의 크기는 선언 시 결정되며 변경 불가능합니다.
- **타입의 구성 요소**: 배열의 크기는 타입 정의의 일부입니다. (크기가 다르면 다른 타입)
- **값 복사(Deep Copy)**: 배열을 대입하면 모든 요소의 값이 복사됩니다.
- **연속된 메모리**: 배열은 메모리 상에 데이터가 빈틈없이 연속적으로 배치됩니다.
