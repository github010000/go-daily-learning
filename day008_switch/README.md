# Day 8: Go switch 문

## 개념 설명

Go 언어의 `switch` 문은 여러 조건 중 하나에 맞는 코드 블록을 실행할 때 사용하는 제어 흐름문입니다. 이는 `if-else if-else` 구조를 대체할 수 있는 강력하고 간결한 대안을 제공합니다. 가장 큰 특징 중 하나는 각 `case` 블록을 실행한 후 **자동으로 해당 `switch` 문을 빠져나간다는 점(자동 `break`)**입니다. 따라서 개발자가 명시적으로 `break`를 사용할 필요가 없어 코드가 훨씬 깔끔해집니다.

만약 개발자가 의도적으로 다음 `case` 블록까지 코드를 실행하고 싶다면(즉, `break`를 건너뛰고 싶다면), **`fallthrough` 키워드**를 사용해야 합니다. `fallthrough`를 사용하면 현재 `case` 블록의 끝에서 바로 다음 `case` 블록의 실행으로 넘어가게 됩니다. 이 동작은 매우 중요하므로 신중하게 사용해야 합니다.

`switch` 문은 단순히 상수를 비교하는 용도로만 사용되지 않습니다. **타입 기반의 `switch` (Type Switch)**를 사용할 수 있습니다. 이 구문은 변수가 어떤 타입을 가지고 있는지 런타임 시점에 확인하고(`interface{}` 타입 변수에 주로 사용), 그 타입에 따라 다른 코드를 실행할 수 있게 해줍니다. 이는 다형성 처리에 매우 유용합니다.

또한, `switch` 문을 **조건문으로 활용**할 수도 있습니다. 이 경우, `switch` 문의 괄호 안에 상수를 넣는 대신 `true`를 넣고, 각 `case` 뒤에 `true`와 같은 불리언 표현식을 넣어 조건을 검사하게 합니다. 이 패턴을 사용하면 여러 개의 복잡한 논리적 조건을 순차적으로 검사할 수 있습니다.

## 코드 설명

1. **기본 Case 분기 (자동 break)**: 가장 일반적인 사용법입니다. `switch` 블록 내에서 `score` 값의 범위를 평가하여 학점을 할당합니다. 값을 평가하는 `case`가 일치하면 해당 블록이 실행되고, 이후 다른 `case`는 무시되며 자동으로 루프를 탈출합니다.

2. **Fallthrough 명시 및 활용**: 요일(1~5)을 입력받아 요일을 표시합니다. 3일과 4일은 `fallthrough`를 사용하여, 3일의 처리가 끝난 후 4일의 처리까지 연달아 실행되도록 구현하여 `fallthrough`의 작동 방식을 보여줍니다.

3. **타입 기반 switch (Type Switch)**: `var data interface{}`에 할당된 변수의 실제 타입을 검사합니다. `switch v := data.(type)` 구문을 통해 변수 `v`를 인터페이스 타입으로 받아, `case string`, `case int` 등을 통해 정확히 어떤 타입인지 판별하고 그에 맞는 코드를 실행합니다.

4. **조건식 switch (Boolean Switch)**: `switch true` 패턴을 사용합니다. `isEven`과 `isPositive`라는 두 변수의 논리값을 조합하여 `case`를 구성합니다. 코드가 순차적으로 평가되기 때문에, 가장 먼저 참이 되는 `case` 블록이 실행됩니다.

## 핵심 포인트
* **자동 `break`**: Go의 `switch`는 조건 일치 후 자동으로 루프를 탈출하므로 명시적인 `break`는 필요 없습니다.
* **`fallthrough`**: 의도적으로 다음 `case`의 실행을 원할 때만 사용하며, 사용하지 않으면 컴파일 오류가 발생하거나 원하는 동작을 하지 않습니다.
* **타입 검사**: `interface{}` 타입 변수와 함께 `switch v := data.(type)` 형태로 사용하여 변수의 런타임 타입을 안전하게 판별할 수 있습니다.
* **조건문 활용**: `switch true` 패턴을 사용하면 `if-else if` 구조와 유사하게 여러 복잡한 논리 조건을 검사할 수 있습니다.
