# Day 5: 타입 변환과 추론

## 개념 설명

Go 언어는 정적 타입 언어이므로, 변수에 할당된 값의 타입은 컴파일 시점에 결정됩니다. 타입 추론(Type Inference) 기능을 통해 `:=` 문법을 사용하면, 컴파일러가 초기화 값으로부터 타입을 자동으로 유추합니다. 예를 들어 `name := "Go"`는 `string`으로, `age := 25`는 `int`로 추론됩니다. 단, `float` 리터럴은 기본적으로 `float64`로 추론됩니다.

명시적 타입 변환은 `T(value)` 형태로 수행됩니다. 예: `float64(10)`, `int(3.14)`. Go는 묵시적 타입 변환을 허용하지 않으므로, 타입을 바꿔야 할 때는 반드시 명시적으로 변환해야 합니다. 예: `int + float64`는 불가능하고, 반드시 둘 중 하나를 다른 타입으로 변환해야 합니다.

특히 **정수형 ↔ 실수형** 간 변환 시 주의가 필요합니다. 실수에서 정수로 변환하면 소수점 이하가 **버려지며**, 큰 정수를 실수형으로 변환할 때는 **정밀도 손실**이 발생할 수 있습니다. 또한, **정수 오버플로우**는 언어 차원에서 보장되지 않으며, 실행 시 정의되지 않은 동작을 유발할 수 있습니다.

Go의 `rune`은 `int32`의 별칭으로 유니코드 코드포인트를 나타내며, `string(r)`을 통해 유니코드 문자로 변환 가능합니다.

## 코드 설명

- **타입 추론 예제**: `name`, `age`, `pi`, `isTrue` 변수는 초기화 값에 따라 각각 `string`, `int`, `float64`, `bool`로 자동 추론됩니다. `%T` 포맷을 사용해 실제 타입을 확인합니다.
- **명시적 타입 변환**: `var y float64 = float64(x)`로 `int` → `float64` 변환을 수행합니다.
- **정수 오버플로우**: `int8` 범위(−128~127)를 벗어나는 계산(`120 + 20`)은 결과가 기대와 달라질 수 있습니다.
- **실수 → 정수 변환**: `int(3.14)`는 `3`으로 소수점 이하가 버려집니다.
- **정밀도 손실**: 매우 큰 `int64` 값을 `float64`로 변환 시, 부동소수점의 정밀도 한계로 인해 값이 왜곡될 수 있습니다.
- **rune ↔ string 변환**: 유니코드 문자 `'한'`을 `rune`으로 저장하고, `string(r)`으로 다시 변환하는 과정을 보여줍니다.

## 핵심 포인트

- **타입 추론**은 `:=`를 통해 자동으로 타입을 유추하며, 초기화 값이 필요합니다.
- **명시적 타입 변환**은 `T(value)` 형태로, Go는 **묵시적 변환을 허용하지 않습니다**.
- **정수 ↔ 실수** 간 변환 시 오버플로우 및 정밀도 손실에 주의하세요.
- **`rune`**은 `int32`로, 유니코드 코드포인트를 다룰 때 유용합니다.
